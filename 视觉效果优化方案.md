# RikkaHub PC 1.1.1 视觉效果优化方案

> 目标：在现有 1.0.0 UI 基础上进行全面的视觉升级，呈现简约、现代、高级感的 AI 应用设计风格。优化范围涵盖色彩/阴影/字体/圆角等设计 token、页面布局、组件形态、交互反馈与动效系统。
> 
> 特殊约定：Claude 主题下对话界面的衬线字体保持不变，其余主题与 UI 区域按本方案统一升级。

---

## 1. 设计目标与原则

### 1.1 目标
- **简约（Minimal）**：减少不必要的装饰线与视觉噪音，让内容本身成为焦点。
- **现代（Modern）**：使用当代桌面 AI 应用的视觉语言：大圆角、柔和分层阴影、呼吸感留白、玻璃拟态点缀。
- **高级感（Premium）**：通过精致的 micro-interaction、克制的动效、一致的间距与字体层级，提升整体质感。
- **一致性（Consistent）**：所有页面、弹窗、侧边栏、输入区遵循同一套设计 token 与交互规范。

### 1.2 原则
1. **内容优先**：降低边框存在感，用背景色阶与阴影建立层级，而非粗边框。
2. **克制动效**：动画必须服务于交互（出现、消失、状态变化），避免炫技式过场。
3. **响应可感知**：按钮点击、输入聚焦、开关切换、消息到达都需要明确的视觉反馈。
4. **明暗一致**：Light / Dark 模式同步升级，切换过程尽量平滑。
5. **可回退**：所有改动按 git commit 原子拆分，任一阶段均可单独回滚。

## 3. 优化范围

### 纳入范围
- `web-ui/app/app.css`：全局设计 token、阴影、圆角、滚动条、关键帧动画。
- `web-ui/app/components/ui/*`：Button、Input、Textarea、Select、Switch、Card、Dialog、Sheet、Tooltip、Skeleton、Spinner 等基础组件视觉升级。
- `web-ui/app/components/conversation-sidebar.tsx`、`components/ui/sidebar.tsx`：侧边栏布局与交互。

- 业务逻辑与后端接口：不改请求/响应逻辑，仅改呈现层。
- 主题色变量名：保持现有 CSS 变量名不变，仅调整值与新增辅助变量。

## 4. 详细设计方案

### 4.1 全局设计 Token 升级

#### 4.1.1 阴影系统（Shadow System）
现有阴影已比 shadcn 默认柔和，但缺少“悬浮感”层级。升级为 4 级阴影：

| Token | 用途 | 建议值（Light） |
|-------|------|----------------|
| `--shadow-float` | 小按钮/徽章/开关 | `0 1px 2px -1px oklch(0 0 0 / 0.08)` |
| `--shadow-card` | 卡片/面板/输入区底座 | `0 4px 12px -2px oklch(0 0 0 / 0.06), 0 2px 4px -1px oklch(0 0 0 / 0.04)` |
| `--shadow-elevated` | 浮层/下拉菜单/Dialog | `0 12px 32px -4px oklch(0 0 0 / 0.1), 0 4px 8px -2px oklch(0 0 0 / 0.05)` |
| `--shadow-strong` | 拖拽中/悬浮预览/强强调 | `0 24px 48px -8px oklch(0 0 0 / 0.12)` |

Dark 模式阴影透明度适度提高，但保持柔和。

#### 4.1.2 圆角系统（Radius System）
在现有 `--radius` 基础上增加语义化圆角变量，确保大卡片与小标签有区分：

| Token | 用途 | 建议值 |
|-------|------|--------|
| `--radius-sm` | 小标签、内部 chip | `calc(var(--radius) - 6px)` |
| `--radius-md` | 按钮、输入框 | `calc(var(--radius) - 2px)` |
| `--radius-lg` | 卡片、面板 | `var(--radius)` |
| `--radius-xl` | 大弹窗、消息气泡 | `calc(var(--radius) + 6px)` |
| `--radius-full` | 胶囊、头像 | `9999px` |

Default 主题 `--radius` 保持 `0.75rem`；可针对主题微调。

#### 4.1.3 间距与层级（Spacing & Elevation）
- 引入 `--surface-1` / `--surface-2` / `--surface-3` 语义化背景变量：
  - `surface-1`：主背景（`--background`）
  - `surface-2`：侧边栏/次级面板（`--sidebar` / `--card`）
  - `surface-3`：悬浮菜单/激活项/输入区背景（`--popover` / `--muted`）
- 卡片内边距统一为 `p-5` / `gap-5`（当前 `gap-6` 偏散）。
- 大区块间距使用 `gap-6` / `gap-8`，避免过度留白。

#### 4.1.4 字体与字重
- UI 字体：保留 Inter / SF Pro Display 优先栈，中文回退 Noto Sans SC / Microsoft YaHei。
- 标题字重：`font-semibold`（600）用于页面标题；`font-medium`（500）用于卡片标题。
- 正文：保持 `font-normal`（400），行高 `leading-relaxed`（1.625）提升阅读舒适度。
- 对话字体：继续通过 `--rikkahub-chat-font` 由用户设置决定；Claude 主题下衬线字体保留。

### 4.2 组件级视觉升级

#### 4.2.1 Button
- **基础**：增加 `active:scale-[0.97]` 按压反馈，`transition-transform` 与 `duration-150`。
- **Default**：背景加细微渐变或更柔和的 hover（`hover:brightness-105`）。
- **Outline / Ghost**：hover 时背景填充更自然，增加 `focus-visible:ring-2`。
- **Icon 按钮**：统一为 `rounded-lg`，hover 增加 `bg-accent/80` 与 `scale-105`。
- **加载状态**：Loader2 旋转 + 按钮整体 `opacity-80`，避免仅图标旋转的生硬感。

#### 4.2.2 Input / Textarea
- **Hover 态**：`hover:border-ring/40`（当前无 hover）。
- **Focus 态**：从 `ring-[3px]` 改为 `ring-2 ring-ring/30`，更精致。
- **Textarea**：增加背景色 `bg-card/50`，减少透明带来的“飘浮”感。
- **Disabled**：背景 `bg-muted/50` + `opacity-60`，区别于普通状态。

#### 4.2.3 Card / Panel
- 增加 `--shadow-card` 作为默认阴影，hover 时轻微上浮 `translateY(-1px)` + 阴影升级。
- 圆角统一使用 `--radius-lg`。
- 卡片标题与描述间距收紧：`gap-1.5`。

#### 4.2.4 Switch
- 增加 thumb 缩放反馈：`data-[state=checked]:scale-110`。
- 背景未选中时保持 `bg-input`，hover 时加深至 `bg-muted-foreground/30`。
- 增加开关切换的弹簧感：使用 `transition-all duration-200 cubic-bezier(0.34, 1.56, 0.64, 1)`。

#### 4.2.5 Dialog / Sheet / Drawer
- Overlay 加深至 `bg-black/40`，增加 `backdrop-blur-sm` 提升层级感。
- Content 增加 `--shadow-elevated`。
- 打开动画：从 `opacity-0 scale-[0.96] translateY(8px)` 到正常状态，时长 250ms，easing `cubic-bezier(0.16, 1, 0.3, 1)`。
- Sheet 侧滑增加 `translateX` 与 overlay fade 同步。

#### 4.2.6 Tooltip / Popover / DropdownMenu
- Tooltip：增加微小 `translateY(2px)` 入场，时长 150ms。
- DropdownMenu / Select：保持 zoom + slide，但增加 `--shadow-elevated`，item hover 增加左侧 2px primary 色指示条（可选）。

#### 4.2.7 Scrollbar
- 细滚动条（6px），hover 时宽度增至 8px，颜色从 `--border` 过渡到 `--muted-foreground/40`。
- WebKit 下使用 `border-radius-full`。

#### 4.2.8 Skeleton / Spinner
- Skeleton：从 `animate-pulse` 升级为扫光 shimmer（类似现有 reasoning shimmer），颜色 `--muted` → `--muted-foreground/20`。
- Spinner：增加 `ease-[cubic-bezier(0.4,0,0.2,1)]`。

### 4.3 页面级视觉升级

#### 4.3.1 聊天主界面（conversations.tsx）
- **顶栏**：
  - 标题栏文字使用 `text-sm font-medium`，模型信息使用更细的 `text-xs text-muted-foreground/70`。
  - 操作按钮（主题切换、系统提示词）统一为 icon-sm ghost，hover 有圆形背景。
  - 顶栏底部分隔线使用 `--divider`，在滚动内容时增加 `shadow-sm` 形成粘性顶栏效果（滚动后显示）。
- **侧边栏**：
  - 会话列表项 hover/active 增加左侧 3px primary 圆角指示条。
  - 新建会话按钮改为更突出的 primary 变体，带 subtle 阴影。
  - 底部工具栏图标统一尺寸与 hover 圆形背景。
- **Workbench 面板**：
  - 面板边界使用 `border-l` + `bg-sidebar/30` 而非纯分割线。
  - 拖拽手柄（ResizableHandle）hover 时加宽并高亮。
- **空状态**：
  - Logo 放大并增加呼吸动画（subtle scale pulse）。
  - 欢迎语使用更大字重与行高，下方快捷操作按钮排列成胶囊网格。

#### 4.3.2 消息气泡（chat-message.tsx）
- **用户消息**：
  - 保持右对齐，背景使用 primary 色系的柔和版本（`bg-primary/90`），文字 `text-primary-foreground`。
  - 圆角：`rounded-2xl rounded-tr-md`（指向发送者的小尾巴视觉）。
  - 阴影：`--shadow-card`。
  - hover 时轻微 `translateX(-2px)` 与阴影加深。
- **助手消息**：
  - 不强调气泡感，使用 `bg-transparent` 或极淡 `bg-muted/30`，让内容更突出。
  - 当存在多 part（图片/工具/推理）时，使用 `bg-card` 卡片包裹，圆角 `rounded-xl`。
  - 头像与名称行固定在最上方，名称使用 `text-xs font-medium text-muted-foreground`。
- **消息操作行**（复制/重试/朗读）：
  - 从始终显示改为 hover 时淡入，避免每条消息都拥挤。
  - 使用 ghost icon-sm，hover 圆形背景。

#### 4.3.3 思维链与 Reasoning Step
- ChainOfThought 展开/收起保留 grid 高度动画，但增加图标旋转 90°。
- Reasoning step shimmer 优化：扫光更细、速度更慢（2s），避免过度抢眼。
- 思考完成后的步骤使用对勾图标 + primary 色，未完成使用脉冲点。

#### 4.3.4 输入区（chat-input.tsx）
- **整体**：使用卡片式容器 `bg-card rounded-2xl shadow-lg border`，悬浮于内容区底部上方。
- **Textarea**：无边框、透明背景、占位符颜色更淡。
- **工具栏**：
  - 模型/推理/搜索/MCP 选择器使用 pill 形状（`rounded-full` + `bg-secondary`）。
  - 发送按钮改为圆形 primary 大按钮（`size-10`），hover 有轻微 scale 与阴影。
  - 停止生成按钮在生成时替换发送按钮，带红色 ring pulse。
- **附件预览**：图片/文件以 chip 形式排列在输入框上方，带删除按钮 hover 显示。

#### 4.3.5 设置页（settings.tsx）
- **侧边栏导航**：
  - 导航项使用 `rounded-lg`，active 状态增加左侧 primary 指示条 + `bg-sidebar-accent`。
  - 图标统一 `size-4`，颜色 `text-muted-foreground`，active 时 `text-sidebar-primary`。
- **主内容区**：
  - 每个设置子页使用卡片分组（`Card` + `CardHeader` + `CardContent`），标题与描述更清晰。
  - 减少全宽输入框的压迫感，表单最大宽度 `max-w-2xl`。
  - 保存/测试按钮固定在卡片右下角或页面底部 sticky bar。
- **视觉层次**：
  - 页面标题 `text-2xl font-semibold`。
  - 卡片标题 `text-base font-medium`。
  - 辅助说明 `text-sm text-muted-foreground`。

#### 4.3.6 图片生成页（images.tsx）
- **左侧边栏**：
  - 模型选择器、数量/比例选择器放入统一的 `Card` 容器。
  - 标题区增加图标与渐变装饰线。
- **主区域**：
  - 提示词输入区改为更现代的大 Textarea，带聚焦时 ring 与轻微阴影变化。
  - 参考图使用 masonry/网格预览，删除按钮 hover 显示。
- **结果展示**：
  - 新生成图片使用 `motion.div` 从 `opacity-0 scale-95` 淡入放大。
  - 加载中使用 Skeleton 卡片占位（与生成数量一致）。
  - 图片 hover 显示下载/删除/查看操作覆盖层。

### 4.4 动效系统

#### 4.4.1 路由/页面过渡
- 在 `root.tsx` 中引入 `AnimatePresence`（来自 `motion`），为 `<Outlet />` 包裹页面级 fade + slide。
- 过渡参数：
  - `initial={{ opacity: 0, y: 8 }}`
  - `animate={{ opacity: 1, y: 0 }}`
  - `exit={{ opacity: 0, y: -4 }}`
  - `transition={{ duration: 0.25, ease: [0.16, 1, 0.3, 1] }}`
- 由于 `home.tsx` 与 `c.$id.tsx` 都渲染同一组件，路由参数变化时不应触发全页重播，仅在 `/` ↔ `/settings` ↔ `/images` 之间过渡。

#### 4.4.2 消息入场动画
- 用户消息：从右侧 `x: 20, opacity: 0` 滑入，时长 250ms。
- 助手消息：从左侧 `x: -10, opacity: 0` 滑入，时长 300ms，带轻微 `scale: 0.98 → 1`。
- 使用 `motion.div` + `layout` 实现列表新增时的自然重排。
- 编辑消息替换时使用 cross-fade。

#### 4.4.3 组件出现/消失
- Dialog/Sheet：见 4.2.5。
- Dropdown/Popover：见 4.2.6。
- Toast：保持 Sonner 默认，但自定义 `toastOptions.classNames` 使用新阴影与圆角。
- TTS Play Bar：从底部 `translateY(100%)` 滑入。

#### 4.4.4 微交互
- 按钮点击：`active:scale-[0.97]`，时长 100ms。
- 输入框聚焦：`ring-2 ring-ring/30` + `border-ring`。
- 开关切换：thumb 带弹簧缩放。
- 侧边栏项 hover：背景从左到右渐变填充（`bg-gradient-to-r`）。
- 滚动到底部按钮：hover 时 `translateY(-2px)` + 阴影加深。

#### 4.4.5 明暗模式切换
- 在 `html` 元素上增加 `transition: color-scheme 0.3s ease, background-color 0.3s ease, color 0.3s ease`。
- 仅对背景色与文字色过渡，避免所有属性过渡导致性能问题。

### 4.5 主题兼容性

- 所有新引入的变量必须在 `:root` 与 `.dark` 中定义默认值。
- 针对每个内置主题（claude / claude-plus / mono / vermillion / amber-mono / mx-brutalist / tiesen / vescrow）检查并调整：
  - 新阴影 token 是否合理（如 mono / amber-mono 主题阴影应接近 0）。
  - 新圆角语义是否与主题原生 `--radius` 冲突。
  - 新组件 hover/active 颜色是否可读。
- 用户自定义主题：由于用户 CSS 可能未定义新变量，必须保证所有新变量都有 fall-back，不破坏旧自定义主题。

### 5.3 性能注意事项
- 消息列表动画使用 `motion.div` 时开启 `layout="position"` 而非完整 `layout`，减少重排。
- 路由动画仅在顶层使用，避免嵌套 AnimatePresence。
- 滚动条样式使用 CSS，不依赖 JS。
- 明暗过渡仅应用于 `background-color` / `color`，避免 `transition-all`。
